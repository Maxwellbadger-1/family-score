## Trust Boundaries

| Boundary | Beschreibung |
|----------|-------------|
| Git Repository | Gitignored Secrets.xcconfig darf diese Grenze nie überschreiten |
| App Target → Widget Extension | Widget Extension erhält KEINEN Zugriff auf Supabase SDK; Daten nur via App Group UserDefaults |
| Apple Developer Portal | App Group Entitlement muss serverseitig im Portal registriert sein; Xcode allein reicht nicht |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | Config/Secrets.xcconfig | mitigate | Secrets.xcconfig ist in .gitignore vor erstem git add; `git check-ignore -v Secrets.xcconfig` als Acceptance Criterion; `git log --all -S "supabase.co"` als End-Verifikation |
| T-1-02 | Information Disclosure | RLS (noch nicht in Plan 01 aktiv) | accept | RLS wird in Plan 03 (Wave 2) aktiviert; ohne Authentifizierung existiert noch kein Family-Scope |
| T-1-03 | Denial of Service | FamilyScoreWidgetExtension Target | mitigate | Supabase SDK wird explizit NICHT im Widget Extension Target verlinkt; Acceptance Criterion prüft `grep Supabase ... | grep widget = 0 Zeilen` |
| T-1-04 | Tampering | App Group Entitlement | mitigate | Beide .entitlements-Dateien erhalten `group.com.familyscore`; Portal-Konfiguration als blocking checkpoint (Task 1); SC-2 Verifikation auf echtem Gerät in Plan 03 |
| T-1-05 | Information Disclosure | Info.plist Build-Setting-Injection | accept | Supabase Anon/Publishable Key ist low-privilege (nur öffentliche API); echter Schutz durch RLS (Plan 03); xcconfig-Pattern hält Key aus Git-History |
