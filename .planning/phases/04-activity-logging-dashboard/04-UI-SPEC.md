## Spacing Scale

Declared values (must be multiples of 4, plus iOS HIG 44pt touch target exception):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, inline padding |
| sm | 8pt | Compact element spacing |
| md | 16pt | Default element spacing |
| lg | 24pt | Section padding |
| xl | 32pt | Layout gaps |
| 2xl | 48pt | Major section breaks |
| touch | 44pt | iOS minimum touch target (Apple HIG) |

Exceptions: `touch` (44pt) is an Apple HIG mandated minimum — not a multiple of 4 but required for all tappable controls.

---

## Typography

All sizes use SF Pro (system font). Two weights only: Regular (400) and Semibold (600).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 17pt | Regular (400) | 1.5 |
| Label | 13pt | Regular (400) | 1.3 |
| Heading | 22pt | Semibold (600) | 1.2 |
| Display | 34pt | Semibold (600) | 1.1 |

Usage guidance:
- **Display** — score numbers inside rings, hero stat (today's total score)
- **Heading** — ring labels (Pflicht, Freizeit, Score), section titles (Heute, Diese Woche), sheet titles
- **Body** — activity list row primary text (title + minutes), form field values
- **Label** — timestamps, category tags, activity list row secondary text, ring percentage captions

## Color

Dark-mode-first palette following Apple Health / Fitness aesthetic.
`systemBackground` and `secondarySystemBackground` adapt automatically between light and dark mode.

| Role | Dark Value | Light Value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | #000000 (systemBackground) | #FFFFFF (systemBackground) | All main surfaces, full-screen backgrounds |
| Secondary (30%) | #1C1C1E (secondarySystemBackground) | #F2F2F7 (secondarySystemBackground) | Cards, sheets, list rows, navigation bar |
| Accent (10%) | #007AFF (systemBlue) | #007AFF (systemBlue) | Primary CTA button, active tab indicator, links |
| Destructive | #FF3B30 (systemRed) | #FF3B30 (systemRed) | Delete entry confirmation button only |

**Semantic ring colors** (additional palette — not part of 60/30/10, used only on ring arcs):

| Ring | Color | iOS Semantic |
|------|-------|-------------|
| Pflicht-Ring (outermost) | #FF3B30 | systemRed |
| Freizeit-Ring (middle) | #34C759 | systemGreen |
| Score-Ring (innermost) | #007AFF | systemBlue |

Ring track (unfilled arc): 15% opacity of the ring's semantic color against the dominant background.

Accent reserved for: primary "Aktivität erfassen" button fill, active bottom tab indicator, navigation link chevrons, interactive toggle active state. Never used for decorative elements or ring fills.

## Ring Layout Contract

Three concentric rings, Apple Activity style, centered on the dashboard screen.

| Ring | Position | Category scope | "Full" threshold |
|------|----------|---------------|-----------------|
| Pflicht | Outermost | Haushalt + Besorgungen + Arbeit/Schule | 60 pts/day |
| Freizeit | Middle | Hobby/Freizeit | 60 pts/day |
| Score | Innermost | All categories combined | 60 pts/day |

- Each ring is a circular arc (0° = 12 o'clock, clockwise fill).
- Progress beyond 100% allowed: arc overlaps itself with a slight opacity reduction on the second lap (consistent with Apple Fitness behavior).
- Gap between adjacent ring tracks: 8pt (sm spacing token).
- Ring stroke width: 20pt (fixed, not a spacing token — hardware-independent feel matching iOS Fitness app).
- Center of rings: displays today's total Score in **Display** typography, label "Punkte heute" in **Label** typography beneath.

## Score & Time Contract

| Parameter | Value |
|-----------|-------|
| Points per minute | 1 (flat ×1 multiplier, all categories) |
| Daily ring full at | 60 pts (= 60 minutes of any tracked activity) |
| Week calculation | Calendar week, Monday reset at 00:00 local time |

## Screen Inventory

### Screen 1 — Dashboard (Tab: Übersicht)

**Purpose:** Show today's ring progress and this week's activity summary per family member.

**Layout:**
- Navigation title: "Übersicht" (large title, inline on scroll)
- Date header below nav: "Montag, 16. Mai" in **Label** typography, systemSecondaryLabel color
- Ring cluster: vertically centered in top 55% of screen
  - Three concentric rings (see Ring Layout Contract)
  - Center text: score number in **Display**, "Punkte heute" in **Label**
  - Below rings: three ring labels in a horizontal stack — each showing ring name + pts — **Label** typography
- Section divider: "Diese Woche" heading in **Heading** typography
- Weekly summary list: one row per family member, showing their Score-Ring progress as a small inline ring + name + pts this week
- FAB (Floating Action Button): "+" SF Symbol, systemBlue fill, 56pt diameter, bottom-right corner, 24pt margin from safe area edges — primary trigger for activity logging

**Empty state (no activities today):**
- Rings display at 0% fill (track-only arcs, full opacity)
- Below rings: heading "Noch keine Aktivitäten heute" in **Heading** typography
- Body: "Tippe auf „+" um deinen ersten Eintrag zu starten." in **Body** typography, systemSecondaryLabel color

---

### Screen 2 — Activity Log Sheet (Modal, triggered from FAB)

**Purpose:** Log a new activity in 3 taps from the dashboard.

**Layout:** `.sheet` presentation (half-screen, drag to dismiss).

**Fields:**
1. **Kategorie** — segmented picker or scroll-select: Haushalt · Besorgungen · Arbeit/Schule · Hobby/Freizeit (4 segments)
2. **Dauer** — inline wheel picker, 5-minute increments, range 5–240 min, default 30 min
3. **Notiz** (optional) — single-line text field, placeholder "Kurze Beschreibung (optional)"

**Controls:**
- Sheet title: "Aktivität erfassen" in **Heading** typography
- Primary CTA button: "Speichern" — full-width, systemBlue fill, 44pt height (touch token), Semibold label
- Cancel: top-left "Abbrechen" text button in systemBlue

**Calculated preview:**
- Below duration picker: "= {N} Punkte" in **Heading** typography, dynamically updating as duration changes

**Error state (save fails):**
"Eintrag konnte nicht gespeichert werden — prüfe deine Internetverbindung und versuche es erneut."
Shown as a `.alert` with single "Verstanden" dismiss button.

---

### Screen 3 — Activity List (Tab: Verlauf)

**Purpose:** Chronological list of all logged activities, grouped by day, with swipe-to-delete.

**Layout:**
- Navigation title: "Verlauf" (large title)
- `List` with section headers per calendar day (e.g. "Montag, 12. Mai")
- Each row:
  - Leading: SF Symbol for category (house.fill / cart.fill / briefcase.fill / gamecontroller.fill), 20pt, ring semantic color
  - Primary: activity category name in **Body** typography
  - Secondary: "{N} min · {N} Punkte" in **Label** typography, systemSecondaryLabel
  - Trailing: timestamp in **Label** typography, systemTertiaryLabel
- Swipe-left action: "Löschen" in systemRed, trash SF Symbol

**Empty state (no activities logged):**
- SF Symbol "tray" (large, systemSecondaryLabel)
- Heading "Noch keine Einträge" in **Heading** typography
- Body "Aktivitäten erscheinen hier, sobald du deinen ersten Eintrag erfasst hast." in **Body** typography, systemSecondaryLabel color

**Delete confirmation** (`.confirmationDialog`):
- Title: "Eintrag löschen"
- Message: "Dieser Eintrag wird dauerhaft gelöscht."
- Actions: "Löschen" (destructive role) · "Abbrechen" (cancel role)

## Interaction & Animation Contract

- Ring fill animation: `.spring(response: 0.6, dampingFraction: 0.75)` on appear and on score update — matches Apple Fitness ring feel
- Sheet presentation: standard `.sheet` — no custom transition
- List row delete: standard `.onDelete` swipe — system animation
- Score update after save: optimistic UI — increment ring locally, confirm from Supabase response; revert on error
- FAB: no animation on tap (iOS convention — sheets self-animate)

---

## Accessibility Contract

- All tappable elements: minimum 44pt touch target (touch spacing token)
- Ring arcs: `accessibilityLabel` = "{Ring name}: {N} von 60 Punkten erreicht ({percent}%)"
- Score center: `accessibilityLabel` = "{N} Punkte heute"
- FAB: `accessibilityLabel` = "Aktivität erfassen"
- List rows: `accessibilityLabel` = "{Kategorie}, {N} Minuten, {N} Punkte, {Uhrzeit}"
- Color is never the sole differentiator — ring position (inner/middle/outer) provides spatial differentiation independent of color
- Dynamic Type: all text uses `.font(.system(size: N))` with `relativeTo:` — or preferably semantic styles where possible
