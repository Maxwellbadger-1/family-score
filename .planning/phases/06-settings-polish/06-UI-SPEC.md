## Spacing Scale

Declared values — all multiples of 4pt (SwiftUI uses points, not pixels):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-label gaps, secondary inline spacing |
| sm | 8pt | Compact spacing between related elements (e.g. role badge + icon) |
| md | 16pt | Default VStack spacing, form field internal padding |
| lg | 24pt | Standard horizontal screen padding, section spacing |
| xl | 32pt | Section separators, generous vertical padding |
| 2xl | 48pt | Hero element top margin (e.g. icon on onboarding screens) |
| 3xl | 64pt | Not used in Phase 6 |

**Source:** Extracted from codebase — LoginView uses `spacing: 16` and `padding(.horizontal, 24)`; CreateFamilyView uses `spacing: 24` and `.padding(.top, 40)`; MemberRow uses `spacing: 12` and `.padding(.vertical, 4)`.

**Exceptions:**
- Child-mode touch targets: minimum 44pt × 44pt for all tappable elements (KID-02). This is an Apple HIG minimum and matches the existing avatar circle size (`frame(width: 44, height: 44)` in MemberRow).
- Settings toggle rows: minimum row height 44pt via `.frame(minHeight: 44)`.
- Point-weight stepper in CategorySettingsView: use SwiftUI `Stepper` native control (meets 44pt touch target automatically).

## Typography

All sizes are SwiftUI semantic font styles mapped to SF Pro. Sizes shown are system defaults at Dynamic Type "Large" (base size).

| Role | SwiftUI Style | Size (base) | Weight | Line Spacing |
|------|--------------|-------------|--------|--------------|
| Body | `.font(.body)` | 17pt | Regular (400) | System default (~1.35) |
| Label / Caption | `.font(.caption)` | 12pt | Regular (400) | System default |
| Subheadline | `.font(.subheadline)` | 15pt | Regular (400) | System default |
| Headline / CTA | `.font(.headline)` | 17pt | Semibold (600) | System default |

**Declared sizes:** 12pt, 15pt, 17pt, 28pt. Four sizes only.

**Declared weights:** Regular (400) and Semibold (600). No other weights.

**Source:** LoginView — button label uses `.font(.headline)`, field labels use `.font(.caption)`, body text uses `.font(.body)`, error text uses `.font(.subheadline)`. Pattern is consistent across all Phase 2–3 views.

**System-automatic styles (not part of the declared scale):**
- `.font(.largeTitle)` (~34pt Bold) — applied to KindDashboardView score number. This is a system style, not a custom declaration; it does not add to the declared size count.
- `.font(.title)` (~28pt Semibold) — applied to KindDashboardView section labels. This maps to the declared 28pt size.
- `navigationTitle` large display — automatically rendered by NavigationStack at ~34pt Bold. Not manually declared.

**Child mode (KID-02):** KindDashboardView uses `.font(.largeTitle)` for the score number and `.font(.title)` for section labels. Both are SwiftUI system styles — no custom `system(size:weight:)` calls. Dynamic Type scaling applies to `.title`; `.largeTitle` is already substantially larger than the body default and is appropriate for child readability without further override.

## Color

All colors use SwiftUI named colors or explicit opacity values. The design is **Dark Mode only** — no light mode adaptation required (Apple Health aesthetic, confirmed in CLAUDE.md: "Dark Mode").

| Role | SwiftUI Expression | Hex Equivalent | Usage |
|------|--------------------|----------------|-------|
| Dominant (60%) | `Color.black` | `#000000` | Full-screen background via `.ignoresSafeArea()` |
| Secondary (30%) | `Color.white.opacity(0.05)` – `0.11` | `#0D0D0D` – `#1C1C1C` effective | List row backgrounds, text field backgrounds, card surfaces |
| Accent (10%) | `Color.white` | `#FFFFFF` | Primary CTA button fill, active text field border, navigation title |
| Destructive | `Color.red` / `Color.red.opacity(0.15)` | `#FF3B30` | Destructive button actions; error banner background at 15% opacity |
| Semantic — Crown / Admin | `Color.yellow` | `#FFD60A` | Admin crown icon only (SF Symbol `crown.fill`) — reuse from MemberRow |
| Semantic — Member Avatar | Avatar hex from profile | Family-defined | Member avatar circle fill — passed from data model |

**Source:** Extracted directly from codebase — LoginView background `Color.black`, field background `Color.white.opacity(0.11)`, CTA button `Color.white`, error banner `Color.red.opacity(0.15)`, MemberRow crown `.foregroundColor(.yellow)`.

**Accent reserved for (exhaustive list):**
- Primary CTA button fill (`Kategorie speichern`, `Einstellungen`, `Modus ändern`)
- Active/focused text field border (opacity 0.45)
- Navigation title text
- Enabled toggle thumb (SwiftUI `Toggle` uses `.tint(.white)` — or system blue if not overridden; see note below)

**Toggle tint note:** SwiftUI `Toggle` on dark background defaults to system green/blue. For visual consistency with the white accent palette, set `.tint(Color.white.opacity(0.9))` on category toggles in CategorySettingsView. This keeps enabled toggles visually consistent without introducing a new accent color.

**Color NOT used for:**
- Success states (use SF Symbol `checkmark.circle.fill` in `.foregroundColor(.green)` sparingly — only for transient confirmations, e.g. "Gespeichert" toast)
- Decorative gradients — none
- Background images or patterns — none

## Component Inventory

These are the new views Phase 6 must build. Each maps to one or more requirements.

### SettingsView (Admin-only root)

- Presented as a tab in MainTabView when `currentMember.role == .admin`
- Style: `NavigationStack` + `List` with `.listStyle(.grouped)` and `.scrollContentBackground(.hidden)`
- Background: `Color.black.ignoresSafeArea()` in `ZStack` behind List
- Sections: "Kategorien", "Mitglieder", "App"
- Admin guard: Tab is not rendered at all when role != `.admin` — no lock screen, simply absent
- **Primary visual anchor:** 'Kategorien' section header — the highest-frequency admin action, appears first in the List.
- **Source pattern:** MemberListView — same List + ZStack + grouped style

### CategorySettingsView

Maps to SETTINGS-01 (category toggle) and SETTINGS-02 (point weight).

**Row layout per category:**

```
[Category icon] [Category name]    [Toggle]
               Point-Gewicht: [Stepper: 0.5 ... 10.0, step 0.5]
```

- Toggle: SwiftUI `Toggle(isOn:)` — `.tint(Color.white.opacity(0.9))`
- Stepper: SwiftUI `Stepper(value:in:step:)` — range 0.5–10.0, step 0.5
- Disabled state: when `is_enabled == false`, the stepper row is hidden (or greyed out with `.opacity(0.4)`) — a disabled category has no meaningful point weight to configure
- Optimistic UI: Toggle and Stepper update local state immediately; Supabase write happens in background Task; error banner appears if write fails
- Row background: `Color.white.opacity(0.05)` via `.listRowBackground()`
- 4 rows: Haushalt, Hobby/Freizeit, Besorgungen, Arbeit/Schule

### MemberSettingsView

Maps to KID-03 (UI mode changeable by Admin).

- Reuses visual pattern from MemberListView (MemberRow sub-view)
- Each row adds a "Modus" picker: `Picker("Modus", selection:)` with `.pickerStyle(.menu)` — options: "Erwachsen", "Kind-vereinfacht"
- Picker is Admin-only (entire view is Admin-only)
- Change triggers `FamilyService.changeMemberRole()` RPC (already implemented in Phase 3)

### KindDashboardView

Maps to KID-02 (child-simplified UI).

- **No TabView** — single NavigationStack
- **No Settings tab**
- **No family comparison** — only current user's data
- Navigation title: "Mein Score" — `.navigationBarTitleDisplayMode(.large)`
- Score ring: single large ring, centered, minimum frame height 200pt (matches RESEARCH.md Pattern 2)
- Activity list: today's own activities only — same row style as adult ActivityListView (Phase 4), but larger row height
- **Large touch targets:** all buttons minimum 44pt × 44pt; prefer 50pt height for primary actions (matches existing CTA button height pattern)
- Font override: Score number displayed at `.font(.largeTitle)` (system-automatic ~34pt Bold) for immediate child readability. Section labels use `.font(.title)` (~28pt Semibold). Both are system styles — no custom `system(size:weight:)` declarations.
- Background: `Color.black.ignoresSafeArea()` — same as all other views

### SettingsView — App Section

- "Datenschutz": NavigationLink to a simple static `PrivacyInfoView` that explains what data the app collects (no external URL needed — App Store review requirement)
- "App-Version": static `Text` showing bundle version — `Bundle.main.infoDictionary?["CFBundleShortVersionString"]`
- No "Sign out" in Settings — sign out stays wherever it was in Phase 4's MainTabView

## Interaction Contracts

### Category Toggle (SETTINGS-01)

| State | Visual |
|-------|--------|
| Enabled | Toggle ON (white tint), stepper row visible |
| Disabled | Toggle OFF, stepper row hidden or `.opacity(0.4)` |
| Saving | `ProgressView()` replaces toggle during async write — or optimistic: toggle updates immediately, spinner shown in row trailing |
| Save error | Error banner at bottom of CategorySettingsView (same `Color.red.opacity(0.85)` HStack pattern as MemberListView) |

**Interaction flow:** Tap toggle → immediate local state update (optimistic) → background `Task { try await categoryService.toggleCategory(...) }` → on error: revert local state + show error banner.

### Point Weight Stepper (SETTINGS-02)

| State | Visual |
|-------|--------|
| Valid range | Stepper with current value label: "1.0×" |
| At minimum (0.5) | Stepper minus button disabled |
| At maximum (10.0) | Stepper plus button disabled |
| Saving | Brief debounce (0.5s) before Supabase write — prevent flood of writes while user taps stepper rapidly |

**Value display format:** `"\(String(format: "%.1f", weight))×"` — e.g. "1.5×"

### Child UI Mode Change (KID-03)

| State | Visual |
|-------|--------|
| Current mode | Picker shows current role label |
| Changing | Picker selection updates immediately; `Task` fires RPC |
| Success | No explicit confirmation — picker reflects new state |
| Error | Error banner in MemberSettingsView |

**No confirmation dialog** for mode change — it is reversible by the admin at any time. Only destructive (irreversible) actions require confirmation dialogs.

### Navigation — Admin Guard

| User Role | Tab Bar Shows |
|-----------|--------------|
| `.admin` | Home, Aktivitäten, Familie, Einstellungen |
| `.adult` | Home, Aktivitäten, Familie |
| `.child` | No TabView — KindDashboardView only |

The Settings tab is conditionally rendered in MainTabView using `if appState.currentMember?.role == .admin`. It is never shown to non-admins — not hidden or locked, simply absent.

## Accessibility

- All interactive elements meet 44pt × 44pt minimum touch target (Apple HIG)
- `.accessibilityLabel()` on icon-only buttons (e.g. toolbar "+" menu in MemberListView pattern)
- Child mode (KID-02): large font, large ring, large buttons — inherently more accessible
- Dynamic Type: use semantic SwiftUI font styles (`.body`, `.caption`, `.headline`) — they scale automatically. The child mode uses `.font(.largeTitle)` and `.font(.title)` which are also system semantic styles and scale with Dynamic Type.
- VoiceOver: SwiftUI Toggle and Stepper provide automatic VoiceOver support. No manual accessibility modifications needed beyond label assignment.
